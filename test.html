03 — Image Placement Specification

Where Every Image Lives

This outreach website is image-forward. Imagery builds emotional credibility and communicates real-world impact. Here is the complete image placement map across all sections.

S2 · Hero Section Dynamic / Auto-rotate

Rotating Hero Backgrounds

  • 5–7 high-resolution full-screen images cycling with Ken Burns (pan & zoom) effect
  • Dark gradient overlay (bottom-up) to keep headline text readable
  • Transition: 1.5s crossfade every 5–6 seconds
  • Images: Communities in outreach, group moments, youth programs, field mission
  • Mobile: images center-cropped, portrait orientation preferred
S3 · Mission/Vision Strip Background Texture

World Map or Abstract Texture

  • Subtle world map silhouette image as section background
  • Low-opacity overlay (15–20%) — just atmospheric, not distracting
  • Warm amber/gold color wash over image
  • Image should evoke "global" without overwhelming the text
S4 · Impact Stats Full-Width Background

Crowd / Community Background

  • Full-width, fixed parallax background image
  • Dark overlay (50–60% opacity) so white stat numbers pop
  • Best image: large group gathering, conference crowd, or community outreach scene
  • Parallax scroll effect on desktop
S5 · Programs Section Card Images ×4

One Image Per Program Card

  • Card 1 — Community Mission: image of field outreach/distribution
  • Card 2 — Mental Health: image of a workshop, circle discussion, or counselling
  • Card 3 — Global Pathway: image of student, travel/international setting
  • Card 4 — Youth Leadership: image of young people, presentation, or forum
  • Hover: image scales up (1.05×), overlay appears with short description
S6 · Global Footprint Map Illustration

Africa / World Map Visual

  • Illustrated or SVG map with highlighted countries (Nigeria, Ghana, Uganda, etc.)
  • Animated dots or pulse markers on each mission country
  • Optionally: 2–3 small inline "field photos" floating near country labels
  • Right panel: country list with flag emoji + brief mission description
S8 · Testimonials Portrait Photos

Individual Portrait per Story

  • Circular or square cropped portrait photo per testimonial
  • If no real portrait available: community/action photo from that mission
  • Carousel: 1 testimonial visible at a time on mobile; 2 on desktop
  • Auto-advance every 5 seconds; manual dots/arrows for control
S9 · Contact Form Split Layout Image

Left-Panel Background Image

  • Left 40% of the section: full-height striking outreach photograph
  • Image overlaid with brand colour (warm amber) at ~40% opacity
  • White text (e.g. quote or tagline) layered on top of image
  • On mobile: image collapses above the form as a banner
S10 · Footer Logo / Background

Brand Logo + Optional Footer Visual

  • Preshol Global logo (light version) in footer top-left
  • Optional: faint world map or subtle dot-pattern background texture
  • Social icons in row: Facebook, Instagram, X/Twitter, LinkedIn, YouTube
04 — Hero Section Deep Dive

Dynamic Hero — Detailed Spec

The hero is the most critical section of this site. It must create immediate emotional connection and communicate mission at a glance.

H1

Background Image Slider

7 full-screen images auto-rotate. Each image gets a subtle Ken Burns animation (slow zoom/pan). Crossfade transition at 1.2s. Duration per image: 5–6 seconds.

7 images Ken Burns Crossfade
H2

Headline Text

Large, bold headline: "Healing Hearts. Empowering Minds. Opening Doors." — layered over the image with a gradient shield behind it. Animated letter-by-letter or word-by-word reveal on first load.

Text reveal Brand voice
H3

Sub-headline + CTA Buttons

One sentence tagline below headline. Two buttons: "Explore Our Work" (primary, filled) and "Partner With Us" (secondary, outlined). Buttons animate in with slight upward float.

Float-in CTA
H4

Slide Indicators

Bottom-center: small dot or dash indicators showing which slide is active. Clicking a dot jumps to that image. On mobile, dots are larger (touch-friendly).

Interactive Accessible
H5

Scroll Indicator

Animated downward-pointing chevron or "scroll" text at the very bottom of the hero. Bounces gently. Disappears after first scroll. Signals content below.

Bounce loop UX hint
H6

Mobile Hero Behaviour

Images are center-cropped and repositioned for portrait. Text size scales down fluidly. CTA buttons stack vertically. Dots remain visible. Minimum height: 90vh on mobile.

Gallery — Technical Behaviour Notes

Filter Tabs: "All", "Nigeria", "Ghana", "Côte d'Ivoire", "Niger", "Uganda", "Conferences" — tabs animate photos in/out with smooth fade + reflow.
Masonry Grid: Desktop: 3–4 columns. Tablet: 2–3 columns. Mobile: 2 columns. Images vary in height for visual richness.
Lightbox: Click any photo → full-screen overlay with image, caption, country label, and left/right arrows to browse within the current filter.
Lazy Loading: Only load images as user scrolls down. This ensures fast initial page load despite the large gallery.

09 — Colour Palette & Typography

Brand Aesthetic Direction

The website should feel warm, authoritative, and human. Not corporate-cold, not overly playful. Earthy and global — inspired by African soil, sunlight, and sky.

Deep Forest #0d2818
Warm Gold #c9963a
Terracotta #b84c2e
Warm Paper #f8f4ee
Sky Blue #2e5f8a
Ink Black #1a1a1a

Display / Headings

Playfair Display — serif. Conveys authority, warmth, and humanity. Use for H1, H2, section titles, and pull-quotes.

Body / UI Text

DM Sans — clean, modern, legible at small sizes. Use for paragraphs, labels, navigation, and form fields.

Monospace Accents

Space Mono — for tags, data labels, counters, and captions. Adds technical precision as a design contrast.

Icon Set

Lucide or Heroicons — clean outline icons for program cards, social links, and UI elements. Consistent stroke weight.

Animation Library

GSAP or CSS transitions. ScrollTrigger for count-up stats, section reveals, parallax effects, and gallery filters.

Image Format

WebP preferred for performance. AVIF as future option. Always include alt text for every image for accessibility and SEO.

10 — Technical & SEO Notes

Build Recommendations

Key technical requirements to ensure the site is fast, findable, and future-proof.

Performance

Lazy-load all gallery images. Compress hero images to max 300KB each. Use image CDN if possible. Target LCP <2.5s.

SEO

Meta title, meta description per page. Open Graph tags for social sharing. Structured data for organisation. Semantic HTML (h1 only once per page).

Accessibility

All images have descriptive alt text. Colour contrast ratio min 4.5:1. Keyboard-navigable. ARIA labels on icons. Skip-to-content link.

Form Handling

Formspree, EmailJS, or backend endpoint. Show success/error state without page reload. Spam protection via honeypot or reCAPTCHA.

Analytics

Google Analytics 4 or Plausible. Track CTA clicks, form submissions, gallery interactions, and scroll depth.

Hosting

Vercel, Netlify, or any static host. Enable HTTPS. Custom domain. Consider next.js or plain HTML/CSS/JS for simplicity and speed.
